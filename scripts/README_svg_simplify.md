# SVG Simplification Tool

Конвертирует сложные SVG (многословные кривые Безье) в упрощённые полигональные
версии. Используется для подготовки SVG-логотипов к рендеру через `3dsvg` /
`SVGLoader` + `ExtrudeGeometry` в three.js.

## Зачем

Пакет `3dsvg` строит 3D-геометрию из SVG-путей через `ExtrudeGeometry`.
Если SVG содержит сотни/тысячи Безье-кривых (например, логотип РУДН —
2459 команд, 13 316 точек после сэмплирования), триангуляция занимает 10+ секунд
и «замораживает» браузер.

## Использование

```bash
cd /app/scripts
npm install simplify-js   # одноразово

# Синтаксис: node simplify_svg_geometrically.cjs <input> <output> [tolerance] [samplesPerCurve]
node simplify_svg_geometrically.cjs \
  /app/frontend/public/rudn-logo-3d.svg \
  /app/frontend/public/rudn-logo-3d-simplified.svg \
  1.5 \
  8
```

## Tolerance ↔ результат

| tolerance | размер | команд  | точек | визуал |
|-----------|-------:|--------:|------:|--------|
| 0.5       | 27 % | 2317    | 1968  | неотличимо от оригинала |
| 1.0       | 23 % | 1989    | 1640  | неотличимо |
| **1.5**   | **20 %** | **1797** | **1448** | **рекомендуется** |
| 2.5       | 16 % | 1569    | 1220  | слегка угловатее в мелких деталях |
| 4.0       | 13 % | 1408    | 1059  | углы заметны на краях |

## Как работает

1. Парсит `d=`-атрибут (команды M/L/C/Q/H/V/A/Z, абсолютные и относительные).
2. Сэмплит каждую Безье-кривую в `samplesPerCurve` точек.
3. Прогоняет полилинию через [simplify-js](https://github.com/mourner/simplify-js)
   (алгоритм Рамера-Дугласа-Пекера).
4. Сохраняет обратно как SVG с командами M/L/H/V/Z (без Безье).

Результат: three.js получает готовый полигон без необходимости сэмплировать
Безье-кривые — триангуляция `ExtrudeGeometry` работает в 5-10× быстрее.
