## 1. 📅 СОВМЕСТНОЕ РАСПИСАНИЕ — ✅ РЕАЛИЗОВАНО

### 💡 Идея
Функция "Расписание вместе" — студенты могут видеть совмещённое расписание с одним или несколькими друзьями, чтобы найти общие окна и договориться о встречах.

**Включается прямо на главном экране** (там же, где обычное расписание) — кнопка-иконка 👥 рядом с датой.

### ✅ РЕАЛИЗОВАНО

#### Backend
- **`POST /api/shared-schedule`** — создание совместного расписания (owner_id + participant_ids)
- **`GET /api/shared-schedule/{telegram_id}`** — получение с расписаниями всех участников и свободными окнами
- **`POST /api/shared-schedule/{id}/add-participant`** — добавление участника
- **`DELETE /api/shared-schedule/{id}/remove-participant/{pid}`** — удаление участника
- **`DELETE /api/shared-schedule/{id}`** — удаление совместного расписания
- **Алгоритм `_compute_free_windows()`** — вычисляет общие свободные окна (мин. 1 час)
- **Модели:** `SharedScheduleCreate`, `SharedScheduleAddParticipant`, `SharedScheduleResponse` в `models.py`
- **Коллекция:** `shared_schedules` в MongoDB
- **Цветовая схема:** 8 предустановленных цветов для участников

#### Frontend — `SharedScheduleView.jsx` (ТАЙМЛАЙН)
- **Визуализация «Таймлайн-лента дня»:**
  - Вертикальная временная шкала 8:00–21:00
  - Колонки по участникам (ваши пары слева, друзья справа)
  - Цветовое кодирование — каждый участник имеет уникальный цвет
  - Зелёные свободные окна с штриховкой и иконкой ☕
  - Красная линия текущего времени с пульсирующей точкой (только сегодня)
  - Sticky заголовки колонок при скролле
  - Авто-скролл к текущему времени при открытии
  - Компактный режим для коротких пар (<1 час)
  - Легенда символов
- **Кнопка-иконка 👥** рядом с кнопкой-датой для переключения режима (indigo при активации)
- **Выбор дня** синхронизирован с основным расписанием (через `selectedDate` из родителя)
- **Friend Picker модалка** — выбор друзей для добавления с скрытием нижнего меню

#### Файлы изменены:
- `frontend/src/components/SharedScheduleView.jsx` — **создан** (таймлайн-визуализация)
- `frontend/src/components/LiveScheduleSection.jsx` — добавлена кнопка режима + conditional rendering

#### `frontend/src/components/StreakRewardModal.jsx` — **ГОТОВ** ✅
Полноценный UI-компонент модального окна награды за серию.

**Реализованные элементы:**
- **Золотой пятиугольник** — скруглённые углы через hardcoded SVG path (quadratic bezier), мгновенный рендер без вычислений в runtime
  - Градиент: `#FAE272 → #E9A422 → #F6CB44 → #C27E08`
  - Свечение по форме пятиугольника через `blur(10px)` SVG path (не прямоугольный border!)
  - Shimmer-полоса через `<motion.rect>` + SVG `clipPath` (работает во всех браузерах)
  - Мягкая тень-эллипс снизу
- **Иконка лаврового венка** — custom inline SVG (левая + правая ветвь + перевязь)
- **Анимированный счётчик** дней (0 → N с easing)
- **Трекер недели** (Mon–Sun): выполненные дни — серый круг с белым венком, будущие — жирные цифры
- **18 частиц-конфетти** при открытии (золотые + белые)
- **Кнопка "Забрать награду"** — pill-shape, чёрная, shimmer + смена на "✓ Получено!" с bounce
- **Кнопка "Не сейчас"** — серый текст под кнопкой
- **Blur-backdrop** (`backdrop-filter: blur(6px)`)
- **Шрифт Poppins** — подключается через Google Fonts динамически, одна вставка в `<head>`

**Props компонента:**
```jsx

  isOpen={bool}
  onClose={fn}

  streakDays={number}       // default: 3

  subtitle={string}         // default: "Ты на правильном пути"
  weekDays={array}          // опционально, формат ниже

```

**Формат `weekDays`:**
```js

  { label: 'Mon', dateNum: 3,  done: true  },

  { label: 'Wed', dateNum: 5,  done: true  },
  { label: 'Thu', dateNum: 6,  done: false },

- **Персонализация:** подставляем имя, кол-во друзей, ближайшие пары

### 📁 Файлы для изменения

### ✅ РЕАЛИЗОВАНО

#### Backend — `scheduler_v2.py`
- **`check_inactive_users()`** — cron-задача каждые 6 часов (CronTrigger `*/6` часов):
  - Обходит всех пользователей с `last_visit_date`
  - Вычисляет `days_inactive = today - last_visit_date`
  - **1 день + стрик ≥ 3:** «🔥 Твой стрик N дней под угрозой!» (+ инфо о щитах)
  - **2 дня + стрик ≥ 3:** «😢 Стрик сгорел...» или «🛡 Щит спас стрик!»

  - **Rate limiting:** `asyncio.sleep(0.1)` между отправками
- **`reset_streak_claimed()`** — cron-задача в 00:05 Москвы, сбрасывает флаг

#### Коллекция MongoDB
- `sent_notifications` — {telegram_id, type, date, created_at} — защита от повторных отправок

- **`POST /api/admin/notifications/parse-telegram`** — парсит публичный TG пост:
  - Извлекает channel + post_id из URL (поддержка `t.me/channel/id` и `t.me/s/channel/id`)

  - Парсит через `BeautifulSoup`: og:title, og:description, og:image
  - Fallback: ищет `tgme_widget_message_text` и `tgme_widget_message_photo_wrap`

  - Если есть `image_url` → `bot.send_photo()` с HTML caption
  - Если нет → `notification_service.send_message()`
  - Rate limiting: `asyncio.sleep(0.05)` между отправками
  - Сохраняет историю в коллекцию `notification_history`
  - Возвращает: `{success, message, sent, failed}`

## 6. 👥 АНИМАЦИЯ ДОБАВЛЕНИЯ В ДРУЗЬЯ

### 💡 Идея
При принятии запроса в друзья — у обоих пользователей появляется кинематографическая анимация: два аватара летят навстречу друг другу, "сливаются" со вспышкой и разделяются снова, оставляя след из частиц.

### 🎨 Анимация (5 фаз)
```
ФАЗА 1: Появление (0-300ms)
  [Твой аватар] ←────────→ [Аватар друга]
  Оба появляются из краёв экрана

ФАЗА 2: Приближение (300-900ms)

### 💡 Идея
Общий чат для ВСЕХ жителей общежития — как WhatsApp группа, но прямо в приложении. Автоопределение общежития по профилю или выбор вручную.

```

│  ┌────────────────────────────┐  │

│  └────────────────────────────┘  │
│                                  │
│                  Ты  14:26       │
│     ┌─────────────────────────┐  │
│     │ Народ, тихий час сегодня│  │

│     └─────────────────────────┘  │
│                                  │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  [📎][😊] Написать сообщение  [➤]│
└──────────────────────────────────┘
```

### 🔧 Архитектура
- **Real-time:** Long Polling (проще и надёжнее без WebSocket)
  - `GET /api/dorm-chat/{dorm_id}/messages?since={timestamp}` — каждые 3 сек
- **MongoDB коллекции:**
  - `dorms` — список общежитий (номер, адрес, кол-во мест)
  - `dorm_members` — кто живёт в каком общежитии
  - `dorm_messages` — сообщения (TTL: 30 дней)
- **Фичи чата:**
  - Реакции на сообщения (❤️ 😂 👍)
  - Ответ на сообщение (reply)
  - Пин важных объявлений (для модераторов)
  - Поиск по чату
  - Прикрепление фото (через Telegram CDN)

### 🔐 Верификация
- При первом входе выбираешь своё общежитие из списка
- Опционально: верификация через номер комнаты + учебный ID

## ⚠️ ВАЖНЫЕ ЗАМЕЧАНИЯ

1. **Streak** — считать только реальные визиты (не API-вызовы), только через Telegram Web App
2. **Совместное расписание** — не хранить копии расписания, только ссылки на `group_id` участников
3. **TG парсинг** — работает только для публичных каналов (без авторизации)
4. **Общажный чат** — TTL на сообщения обязателен (MongoDB TTL index) для экономии места
5. **Шаринг** — `html2canvas` плохо работает с кастомными шрифтами, использовать системные или `Pillow` на бэкенде
