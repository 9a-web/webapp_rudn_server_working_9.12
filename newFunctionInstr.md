## 1. 📅 СОВМЕСТНОЕ РАСПИСАНИЕ

### 💡 Идея
Функция "Расписание вместе" — студенты могут видеть совмещённое расписание с одним или несколькими друзьями, чтобы найти общие окна и договориться о встречах.

**Включается прямо на главном экране** (там же, где обычное расписание) — переключатель режима отображения.

### 🎨 UX/UI Дизайн
```
┌─────────────────────────────────────┐
│  🗓 Совместное расписание            │
│  ┌──────┬──────┬──────┐             │
│  │ Пн   │ Вт   │ Ср   │             │
│  ├──────┼──────┼──────┤             │
│  │ [Ты] │[Ты] │      │  ← цветные  │
│  │[Саша]│     │[Саша]│    полоски   │
│  │      │[Ты] │[Ты]  │             │
│  │ 🟢   │      │ 🟢   │  ← общее   │
│  │ ОКНО │      │ ОКНО │    время    │
│  └──────┴──────┴──────┘             │
│                                     │
│  👥 Участники: [+] Добавить друга   │
│  [Ты 🔵] [Саша 🟣] [Маша 🟡]       │
└─────────────────────────────────────┘
```

### 🔧 Логика
- **Хранение:** новая коллекция `shared_schedules` с полями `owner_id`, `participants[]`, `group_ids[]`
- **Алгоритм:** загружаем расписания всех участников → находим общие "окна" (свободные промежутки) → выделяем зелёным
- **Цветовая схема:** каждый участник получает уникальный цвет (аватар + имя), занятия отображаются полупрозрачными полосками
- **Интерактив:** тап на "зелёное окно" → предложить создать событие / написать в чат

### ✨ Анимации
- При добавлении участника: аватар "влетает" снизу с bounce-эффектом
- Общие окна пульсируют зелёным свечением
- Плавный переход при переключении недель (slide)

## 2. 🔥 STREAK-МЕХАНИКА

### 💡 Идея
Ежедневная серия посещений приложения — "огненная" страйк-система как в Duolingo, но с РУДН-тематикой (книги, звёзды, шапка выпускника).

> **Компонент `StreakRewardModal.jsx` — уже реализован** ✅

### 🎨 UX/UI Дизайн
```
┌─────────────────────────────┐
│       🔥 15 дней подряд!     │
│                             │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  Пн  Вт  Ср  Чт  Пт  Сб  Вс│
│  ✅  ✅  ✅  ✅  ✅  ✅  🔵  │
│                             │
│  🏆 Следующая цель: 30 дней │
│  ████████████░░░░░ 15/30    │
│                             │
│  🛡 Щит заморозки: 1 шт.    │
└─────────────────────────────┘
```

### 🔧 Логика
- **Новые поля в `user_stats`:**
  - `visit_streak_current` — текущая серия (дней)
  - `visit_streak_max` — рекорд за всё время
  - `last_visit_date` — дата последнего визита (YYYY-MM-DD)
  - `freeze_shields` — кол-во щитов заморозки (начисляются за каждые 7 дней стрика)
  - `streak_claimed_today` — флаг: уже показан попап сегодня
- **Начисление:** при открытии приложения — `POST /api/users/{id}/visit`
  - Сегодня уже заходил → ничего
  - Вчера заходил → `+1` страйк, если `freeze_shields > 0` — тратить не нужно
  - Пропуск 1 дня + щит есть → страйк сохраняется, `freeze_shields -= 1`
  - Пропуск 1 дня без щита → страйк обнуляется
- **Милестоны:** 3, 7, 14, 30, 60, 100, 365 дней → `StreakRewardModal` + новое достижение
- **Щиты заморозки:** 1 щит за каждые 7 дней страйка

### ✨ Анимации (уже в StreakRewardModal)
- Огонь: CSS keyframes (flickering flame)
- При рекорде: конфетти + большая огненная анимация
- Цифра страйка "прыгает" при обновлении (bounce scale)
- Calendario недели: дни "заполняются" чекмарком с pop-анимацией

### 📁 Файлы для изменения
- `backend/server.py` — endpoint `POST /api/users/{id}/visit`
- `backend/models.py` — поля в `UserStats`
- `frontend/src/App.jsx` — вызов endpoint при запуске + показ `StreakRewardModal`
- `frontend/src/components/StreakRewardModal.jsx` — **уже готов** ✅
- `frontend/src/components/DesktopSidebar.jsx` — виджет стрика

## 3. ⏰ АВТОМАТИЧЕСКИЕ НАПОМИНАНИЯ О ВОЗВРАЩЕНИИ

### 💡 Идея
Умные Telegram-уведомления, которые появляются когда пользователь давно не заходил, с персонализированным текстом и мотивацией.

### 🎨 UX/UI — Типы уведомлений
```
📱 Telegram уведомления:

После 1 дня (страйк под угрозой):
🔥 "Эй! Твой стрик 12 дней под угрозой!
    Зайди в приложение — сохрани свою серию"
    [🚀 Открыть приложение]

После 2 дней (страйк сгорел):
😢 "Твой стрик 12 дней сгорел...
    Но всё можно начать заново! Сегодня день 1"
    [💪 Начать снова]

После 7 дней (мягкое напоминание):
🌟 "Привет, [Имя]! Давно не виделись.
    Завтра у тебя 3 пары — проверь расписание?"
    [📅 Посмотреть расписание]

После 30 дней (персональное):
❓ "Всё в порядке? Ты не заходил уже месяц.
    Твои друзья [Саша, Маша] уже ждут 👋"
    [👥 Открыть]
```

### 🔧 Логика
- **Новые задания в `scheduler_v2.py`:**
  ```python
  INACTIVITY_1D  # 1 день  → предупреждение о страйке
  INACTIVITY_2D  # 2 дня   → страйк потерян
  INACTIVITY_7D  # неделя  → мягкое напоминание + расписание
  INACTIVITY_30D # месяц   → персональное + упоминание друзей
  ```
- **Поле в `user_settings`:** `last_activity` (уже существует) → читаем из него
- **Персонализация:** подставляем имя, кол-во друзей, ближайшие пары
- **Частота:** Проверка каждые 6 часов через APScheduler

### 📁 Файлы для изменения
- `backend/scheduler_v2.py` — новые задания inactivity check
- `backend/notifications.py` — новые шаблоны текстов

## 6. 👥 АНИМАЦИЯ ДОБАВЛЕНИЯ В ДРУЗЬЯ

### 💡 Идея
При принятии запроса в друзья — у обоих пользователей появляется кинематографическая анимация: два аватара летят навстречу друг другу, "сливаются" со вспышкой и разделяются снова, оставляя след из частиц.

### 🎨 Анимация (5 фаз)
```
ФАЗА 1: Появление (0-300ms)
  [Твой аватар] ←────────→ [Аватар друга]
  Оба появляются из краёв экрана

ФАЗА 2: Приближение (300-900ms)
  [Твой аватар] →→→→→→→→→→ [Аватар друга]
  Летят навстречу, вращаясь немного
  ✨ Искры появляются на пути

ФАЗА 3: Столкновение (900-1100ms)
  ══════════[💥 ВСПЫШКА]══════════
  Оба аватара сжимаются → белая вспышка
  Сердечко ❤️ появляется в центре

ФАЗА 4: Разделение (1100-1500ms)
  [Твой аватар] ←←←←← 💫 →→→→→ [Аватар друга]
  Разлетаются с частицами/конфетти

ФАЗА 5: Итог (1500-2500ms)
  [Твой аватар] 👫 [Аватар друга]
  "Вы теперь друзья!" надпись с pop-анимацией
  Счётчик друзей анимированно увеличивается
```

### 🔧 Реализация
- **Библиотека:** `framer-motion` (уже в проекте) + CSS keyframes
- **Компонент:** `FriendAcceptedAnimation.jsx` — fullscreen overlay с `z-index: 9999`
- **Триггер:** при принятии запроса в друзья вызвать из `FriendsSection.jsx`
- **Canvas particles:** небольшой particle engine для искр/конфетти

### 📁 Файлы для изменения
- `frontend/src/components/FriendAcceptedAnimation.jsx` — новый компонент (создать)
- `frontend/src/components/FriendsSection.jsx` — подключить анимацию при принятии запроса

### 🔧 Архитектура
- **Real-time:** Long Polling (проще и надёжнее без WebSocket)
  - `GET /api/dorm-chat/{dorm_id}/messages?since={timestamp}` — каждые 3 сек
- **MongoDB коллекции:**
  - `dorms` — список общежитий (номер, адрес, кол-во мест)
  - `dorm_members` — кто живёт в каком общежитии
  - `dorm_messages` — сообщения (TTL: 30 дней)
- **Фичи чата:**
  - Реакции на сообщения (❤️ 😂 👍)
  - Ответ на сообщение (reply)
  - Пин важных объявлений (для модераторов)
  - Поиск по чату
  - Прикрепление фото (через Telegram CDN)

### 🔐 Верификация
- При первом входе выбираешь своё общежитие из списка
- Опционально: верификация через номер комнаты + учебный ID

## ⚠️ ВАЖНЫЕ ЗАМЕЧАНИЯ

1. **Streak** — считать только реальные визиты (не API-вызовы), только через Telegram Web App
2. **Совместное расписание** — не хранить копии расписания, только ссылки на `group_id` участников
3. **TG парсинг** — работает только для публичных каналов (без авторизации)
4. **Общажный чат** — TTL на сообщения обязателен (MongoDB TTL index) для экономии места
5. **Шаринг** — `html2canvas` плохо работает с кастомными шрифтами, использовать системные или `Pillow` на бэкенде
